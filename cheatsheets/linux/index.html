    How to configure a hostname on a Linux

    Modify the system's name temporarily

    hostname demo.example.com

    Modify the system's name permanently

    hostnamectl set-hostname demo.example.com

    You can confirm this entry by displaying the /etc/hostname file contents.

    Set static ip address

    Check what your network manager:

    ps aux | grep -E 'dhcpcd|NetworkManager|systemd-networkd'

    DHCPCD

    Edit the file in /etc/dhcpcd.conf

    Add this section :

    interface wlan0
static ip_address=192.168.x.x/24
static routers=192.168.x.x

    Restart or reboot:

    sudo systemctl restart dhcpcd

    NetworkManager

    Using nmcli :

    nmcli connection show
nmcli connection modify <YourWifiConnection> ipv4.method manual ipv4.addresses 192.168.x.x/24 ipv4.gateway 192.168.x.x
nmcli connection modify <YourWifiConnection> ipv4.dns-priority -1
nmcli connection down <YourWifiConnection> && nmcli connection up <YourWifiConnection>

    Mannualy :

    Edit the file in /etc/NetworkManager/system-connections/<YourWifiProfile>

    In the [ipv4] section:

    [ipv4]
method=manual
address1=192.168.x.x/24,192.168.x.x

    Save and apply:

    sudo chmod 600 /etc/NetworkManager/system-connections/MyWifiConnection.nmconnection
sudo systemctl restart NetworkManager

    Manual configuration (on Ubuntu-based systems)

    Edit the network configuration:

    sudo nano /etc/network/interfaces

    Add or modify the following lines:

    allow-hotplug ethX
iface ethX inet static
    address 192.168.x.x
    netmask 255.255.255.0
    gateway 192.168.x.x

    Restart networking:

    sudo systemctl restart networking

    File permissions

    File type ; User ; Group ; Everyone

    File types :

    • d Directory
    • - Regular file
    • l Symbolic Link

    Permissions digits :

    • r (read): 4
    • w (write): 2
    • x (execute): 1

    Change folder owner

    chown -R <user>:<group> <folder_path>

    Configure SSH key authentification based

    Make sure your server have the file .ssh/authorized_keys in the home of the user.

    Generate a key for the client :

    ssh-keygen -t ed25519 -C "your_email@example.com"

    You can then use ssh-copy-id [-i <key.pub>] <user>@<server_ip> to copy the key to the server authorized_keys

    In the server : Edit /etc/ssh/sshd_config:

    And change these :

    ...
PubkeyAuthentication yes
...
AuthorizedKeysFile      .ssh/authorized_keys .ssh/authorized_keys2
...
PasswordAuthentication no
...

    SSL

    Generate ssl key

    • Key generation : genpkey
    • Certificat request : req (-x509)
    • Certificat signing : x509
    • Verify certification chain : verify

    Note: Since 2016, Firefox has disabled the Domain Name; it must now be done with a SAN (v1).

    Model generation v3.

    RCA

    Generate RCA key :

    openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:4096 -out rca.key

    Create RCA configuration :

    [req]
prompt = no
distinguished_name = dn
req_extensions = ext

[dn]
CN = <Commun Name>
O = <Organization>

[ext]
subjectAltName = DNS:*.<my-domain>

    NB : You can precise DNS or IP

    Generate RCA certificat :

    openssl req -x509 -key rca.key -out rca.crt -days 365 -config conf_rca

    Check certificat:

    openssl x509 -nout -text -in rca.crt

    ICA (Intermediate CA)

    Generate ICA key :

    openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:4096 -out ica.key

    Create ICA configuration :

    [req]
prompt = no
distinguished_name = dn
req_extensions = ext
x509_extensions = ext

[dn]
CN = <Commun Name>
O = <Organization>

[ext]
basicConstraints = CA:true
subjectAltName = DNS:*.<my-domain>

    On a ajouter x509_extensions = ext

    Generate ICA certificat :

    openssl req -x509 -key ica.key -out ica.crt -CA rca.crt -CAkey rca.key -days 365 -config conf_ica

    -CA : auto qui va signer ce cert

    Vertifier que la chaine est valide :

    openssl verify -CAfile rca.crt ica.crt

    Check certificat:

    openssl x509 -nout -text -in ica.crt

    CA Web

    Generate CA key :

    openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:4096 -out web.key

    Create CA configuration :

    [req]
prompt = no
distinguished_name = dn
req_extensions = ext
x509_extensions = ext

[dn]
CN = <Commun Name>
O = <Organization>

[ext]
basicConstraints = CA:false
subjectAltName = DNS:web.<my-domain>

    Generate CA certificat :

    openssl req -x509 -key web.key -out web.crt -CA ica.crt -CAkey ica.key -days 365 -config conf_web

    Verify full chain:

    cat ica.crt rca.crt > chain.crt
openssl verify -CAfile chain.crt web.crt
