    Radare2

    Basic Debugging Workflow

    1. Start Radare2 in Debug Mode

    r2 -d <binary>

    Launches Radare2 with the specified binary in debug mode (similar to gdb ./binary).

    2. Analyze the Binary

    aaa

    Performs auto analysis of the binary — finds functions, basic blocks, symbols, etc. You can also use:

    • aa — basic analysis
    • aaa — deeper analysis (recommended for most cases)

    3. Enter Visual Mode

    v

    Enters visual mode for interactive inspection of code and memory. Use these shortcuts inside visual mode:

    • p — change view (disassembly, hex, graph, etc.)
    • V — visual graph mode
    • q — quit visual mode

    4. Inspect Functions and Code

    pdf

    Prints the disassembly of the current function.

    You can also use:

    pd <n>

    to print <n> instructions from the current address.

    5. Set Breakpoints

    db <address>

    Sets a breakpoint at the specified address. Example:

    db main

    sets a breakpoint at the main() function.

    6. Continue Execution

    dc

    Continue program execution until the next breakpoint or program termination.

    Other useful commands:

    • ds — step into (like stepi in GDB)
    • dso — step over (skip function calls)
    • dr — show registers
    • px — dump memory in hex
    • pxw — dump memory as 32-bit words

    7. Common Visual Mode Shortcuts

    Key Description
    v Enter visual mode
    V Visual graph mode
    p Change display mode
    s Seek (move to address)
    / Search pattern
    db Set breakpoint
    dc Continue execution
    q Quit visual mode
