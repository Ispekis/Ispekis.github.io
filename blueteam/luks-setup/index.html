    Luks

    Partition the disk

    fdisk /dev/sdX

    Useful fdisk commands:

       d   delete a partition
   F   list free unpartitioned space
   l   list known partition types
   n   add a new partition
   p   print the partition table
   t   change a partition type
   v   verify the partition table
   i   print information about a partition
   e   resize a partition
   T   discard (trim) sectors

    Format the partition

    mkfs.ext4 -b 4096 /dev/sdX

    Mount the partition

    mount /dev/sdX <directory>

    Get UUID

    lsblk -f
# or
blkid /dev/sdX

    LUKS Setup

    Install required packages

    sudo apt install lvm2 cryptsetup cryptsetup-initramfs systemd-cryptsetup

    Create the keyfile

    sudo mkdir -p /etc/luks
sudo dd if=/dev/random of=/etc/luks/luks.key bs=1 count=256

    Secure the keyfile and directory — root only, immutable: Make the file immutable

    Format the LUKS partition

    Without keyfile (password prompt):

    cryptsetup --verbose luksFormat /dev/sdX

    With keyfile:

    cryptsetup --verbose --key-size=256 luksFormat /dev/sdX /etc/luks/part2.key

    Add keyfile to existing LUKS partition

    cryptsetup luksAddKey /dev/sdX /etc/luks/part2.key

    Verify LUKS partition

    cryptsetup isLuks /dev/sdX && echo "Valid LUKS partition"

    Open (decrypt) the LUKS partition

    Manually with password:

    cryptsetup luksOpen /dev/sdX crypt_sdX

    Automatically with keyfile:

    cryptsetup luksOpen --key-file /etc/luks/part2.key /dev/sdX crypt_sdX

    Close (re-lock) the LUKS partition

    umount /data_encrypted
cryptsetup luksClose crypt_sdX

    /etc/crypttab

    Tells the system to decrypt the partition automatically at boot using the keyfile.

    # <name>        <device>                                    <keyfile>               <options>
crypt_sdX      UUID=<uuid-of-sdX>                         /etc/luks/part2.key     luks

    Replace <uuid-of-sdX> with the UUID from lsblk -f or blkid /dev/sdX.

    | Field | Description |
|-------|-------------|
| `name` | The mapper name — decrypted device will appear as `/dev/mapper/<name>` |
| `device` | The raw encrypted partition, identified by UUID (recommended) or path like `/dev/sdX` |
| `keyfile` | Path to the keyfile used to decrypt. Use `none` to prompt for a password at boot |
| `options` | Comma-separated options (see below) |

    crypttab options

    Option Description
    luks Specifies the device is a LUKS-formatted partition (recommended, enables LUKS-specific handling)
    plain Use plain dm-crypt instead of LUKS (no header, no metadata)
    noauto Do not decrypt automatically at boot — must be opened manually
    noearly Decrypt late in the boot process, after mounts (useful for non-root partitions)
    readonly Open the device in read-only mode
    discard Enable TRIM/discard passthrough (improves SSD performance, slight security trade-off)
    tries=N Number of password attempts before failing (default: 3). Use tries=0 for unlimited
    timeout=N Seconds to wait for a password before giving up
    keyfile-timeout=N Seconds to wait for the keyfile to become available (useful for keyfiles on USB drives)
    swap Format the device as swap after decryption (data is not preserved across reboots)
    tmp Format the device as a tmpfs after decryption (data is not preserved across reboots)

    /etc/fstab

    Tells the system to mount the decrypted partition automatically at boot.

    # <device>                      <mountpoint>        <type>  <options>   <dump>  <pass>
/dev/mapper/crypt_sdX          /<directory>     ext4    defaults    0       1

    | Field | Description |
|-------|-------------|
| `device` | The decrypted mapper device from crypttab — always `/dev/mapper/<name>` |
| `mountpoint` | Directory where the partition will be mounted |
| `type` | Filesystem type: `ext4`, `xfs`, `btrfs`, etc. Use `auto` to detect automatically |
| `options` | Mount options (see below) |
| `dump` | Used by `dump` backup tool. Almost always `0` (disabled) |
| `pass` | fsck check order at boot: `0` = skip, `1` = root partition only, `2` = check after root |

    fstab options

    Option Description
    defaults Shorthand for rw,suid,dev,exec,auto,nouser,async — suitable for most use cases
    rw / ro Mount read-write or read-only
    noauto Do not mount automatically at boot — must be mounted manually with mount -a
    nofail Do not fail boot if the device is missing or cannot be mounted
    noexec Prevent execution of binaries on this partition (good for data-only partitions)
    nosuid Ignore setuid/setgid bits (improves security on non-system partitions)
    nodev Prevent device files from being interpreted (recommended for non-root partitions)
    user Allow any user to mount the partition (implies noexec, nosuid, nodev)
    uid=N / gid=N Force ownership to a specific user/group ID (mainly for FAT/NTFS)
    umask=N Set default permissions mask (mainly for FAT/NTFS)
    discard Enable TRIM passthrough for SSDs (must also be enabled in crypttab)
    x-systemd.requires= Wait for a specific systemd unit before mounting (e.g. the cryptsetup service)
